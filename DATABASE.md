## Purpose: Explain information storage. This document must be complete before any database schema is created.

## Minimum required content:
• Important entities — all major data objects and their attributes
• Relationships — how entities relate to each other with cardinality
• Data flow — what gets created, read, updated, and deleted in each workflow
• Scaling considerations — how the data model supports growth

Design for future growth, not only today's features. A schema that cannot evolve without breaking the application is a design failure
